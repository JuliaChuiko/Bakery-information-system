-------------Проектирование и создание информационной системы Мини-пекарни-----------

Цель: создать инф. систему для управления продажами в пекарне.
Пользователи системы: сотрудники и владелец сети пекарен.
Реализация выполнена с использованием языка C# и СУБД PostgreSQL. Архитектура системы соответствует шаблону MVC.

Выбрана трехуровневая архитектура:клиентское приложение ("тонкий клиент"),сервер приложений,
сервер базы данных.

1.слой клиента - его основная функция представление задач и результатов, понятных пользователю.

2.слой логики - координирует программу, обрабатывает команды, выполняет логические решения,вычисления,расчеты.

3.слой данных - хранит информацию, извлекаемую из БД.

Для Back-end части использовались фреймворк - ASP .NET Core и библиотека Npgsql.
Для Front-end части - Bootstrap 3.
В качестве веб-сервера взят - IIS, так как отлично совместим с программной платформой .Net Framework.
Среда разработки: Visual Studio 2019.

Для данной информационной системы выбрана объектно-реляционная БД - PostgreSQL, так как она позволяет обрабатывать большие объемы данных, создать свой собственный тип данных. Имеет много функций обеспечения целостности, которые дают уверенность, что будут сохранены только корректные данные.PostgreSQL не имеет требований к оборудованию, работает на всех основных ОС.


Согласно выбранной архитектуре, программа разделена на 3 основные части: модели, контроллеры и представления.

HomeController -  отвечает за аутентификацию и авторизацию.
OwnerController -  отвечает за запросы/действия владельца сети пекарен.
BakerController -  отвечает за запросы/действия пекаря/кондитера.
SellerController - отвечает за запросы/действия продавца.
AdminController  - отвечает за запросы/действия системного администратора.

Для безопасности осуществлено разбиение на групповые роли: владелец, продавец, пекарь/кондитер. В зависимости от занимаемой должности, пользователь добавляется к групповой роли, наследуя ее права.
Аккаунты всех сотрудников сети пекарен можно найти в отдельной базе "Database_for_Users" в таблице "UsersAccounts2". 
Пароли зашифрованы.

---------------------------------------------------------------------
Database_for_Users.backup - БД с аккаунтами пользователей.
Database_for_bakery.backup - БД пекарни.
